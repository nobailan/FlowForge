# FlowForge v0.6 — 实时监控

> **发布日期**: 2025-06-25  
> **核心主题**: 真正的实时流式监控 —— 每个节点的思考过程、工具调用、完成状态即时可见  

---

## 一、v0.6 做了什么

v0.4-v0.5 的"实时监控"实际上只是**执行完成后批量回放**。MonitorBridge 虽然在线程中收集 SSE 事件，但 BridgePoller 从未被接入——原因是一个跨 asyncio event loop 的技术障碍。

v0.6 解决了这个从项目初期就存在的问题。

### 技术方案：队列桥接

```
OpenCode SSE 事件流
    │ (独立线程)
    ▼
MonitorBridge ──→ queue.Queue (线程安全全局队列)
                      │
                      ▼ (FastAPI event loop, 150ms 轮询)
                 stream_pusher ──→ ws_manager.broadcast_streaming()
                                        │
                                        ▼ (WebSocket)
                                   前端 addEvent()
```

**关键点**：WebSocket 的 `send_json()` 必须在 FastAPI 的 event loop 中执行。v0.4-v0.5 的 BridgePoller 在自己的 `asyncio.run()` loop 中直接调 WebSocket → 跨 loop 操作失败。v0.6 改为 MonitorBridge 只负责写队列，推送由 FastAPI 侧的 `stream_pusher` 协程完成。

## 二、改动文件

```
backend/src/
├── adapter/
│   ├── monitor_bridge.py    # 重写：全局 _stream_queue + 5 种标准化事件

│   └── execute.py           # 新增 stream_pusher 协程（150ms 轮询推送）
├── engine/
│   ├── state.py             # AgentState 新增 execution_id 字段
│   └── executor.py          # 初始化 state 时传入 execution_id

├── hooks/
│   └── useExecutionConsole.ts  # 新事件格式 + 空文本过滤 + Token 防重复
└── components/Monitor/
    └── ExecutionConsole.tsx    # 轮询完成后立即停止（修复 Token 无限累加）
```

## 三、标准化事件类型

| 事件 | 触发时机 | 关键字段 |
|------|---------|---------|
| `node:start` | 节点开始执行 | `{node_id, node_type}` |
| `node:thinking` | Agent 输出文本增量 | `{node_id, text}` |
| `node:tool` | 工具调用开始 | `{node_id, tool_name, tool_input}` |
| `node:tool_result` | 工具调用结束 | `{node_id, summary}` |
| `node:end` | 节点执行结束 | `{node_id, status, tokens, latency_ms, output_preview}` |

## 四、附带修复

| Bug | 描述 | 修复 |
|-----|------|------|
| Token 无限累加 | 轮询每 3 秒重新加一次所有节点的 token | 完成即停轮询 + 首次完成才累加 |
| execution_id = "default" | state 缺少该字段，事件无法匹配 | 加入 AgentState 并透传 |
| 空 thinking 刷屏 | SSE 空文本 delta 被渲染成空白行 | 后前端双重过滤 |
| `import time` 缺失 | execute.py 新增代码用了 `time.time()` 但未导入 | 补充导入 |

## 五、验收效果

- 节点开始执行 → 控制台立即显示蓝色脉冲，状态从"等待中"变为运行中

- 工具调用 → 工具名和参数实时显示
