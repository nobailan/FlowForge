# FlowForge v0.5 说明

> **发布日期**: 2025-06-25  
> **核心主题**: Token 优化 —— 四刀砍掉 80% 不必要消耗  

---

## 一、v0.5 做了什么

v0.4 的多 Agent 协作流程存在严重的 Token 浪费问题：每个节点都是全工具 OpenCode Agent，上游把完整文件内容塞给下游，输出啰嗦冗长。v0.5 引入四项优化：

| 刀 | 名称 | 手段 | 效果 |
|----|------|------|------|
| 🗄️ | 工件存储 | Redis 缓存文件内容，Agent 间只传 `[REF: artifact://key]` 引用 | 文件传递 15,000T → 500T |
| 📜 | 压缩摘要 | System Prompt 强制 JSON 输出，拒绝啰嗦文本 | 单节点输出 800T → 200T |
| 🛠️ | 按需加载 | 根据节点角色和任务关键词，只挂载必需工具 | 工具定义 800T → 150-450T |
| 🔄 | 滑动窗口 | 裁剪对话历史，只保留任务目标 + 最近 K 轮 | 长对话节省 40-60% |

## 三、Token 消耗对比

### 实测数据（Parallel Experts 模板，4 个 Agent 节点）

| 指标 | v0.4 | v0.5 | 节省 |
|------|------|------|------|
| 工具定义 overhead | 800T/节点 × 4 | 250T/节点 avg × 4 | -69% |
| 输出啰嗦度 | 500T/节点 | 200T/节点 (JSON) | -60% |
| Token 虚高 bug | 有（double-count） | 已修复 | 准确计数 |
| 文件传递（实测） | 10,000T（值传递） | 300T（REF 引用） | **-97%** |
| **单次执行典型** | **~32,000T** | **~20,000T** | **~38%** |

### 带文件读取的真实场景

```
v0.4: 40,000T → v0.5: 8,000T ≈ 节省 80%
```

## 五、关键设计决策

### 1. 工件存储用 Redis
- 部署简单：Windows 单文件 `redis-server.exe`
- Key 格式：`artifact:{execution_id}:{node_id}:{filename}:v{version}`
- 自动 TTL 10 分钟，执行结束 cleanup
- 引用语法：`[REF: artifact://key]` / `[REF: artifact://key#L10-L50]`
- Redis 不可用时自动回退内存存储

### 2. 工具按需分配
- Dispatcher / Aggregator / Router → `allowed_tools: []`（零工具）
- Worker 子类型分类：`reader_worker` (glob/grep/read) / `writer_worker` (read/write/bash) / `code_worker` (read/write/bash/grep) / `researcher` (web_search/web_fetch/read)
- 关键词匹配 + 优先级判定

### 3. JSON 强制输出
- 每个节点 System Prompt 末尾注入 JSON Schema
- Dispatcher 输出 `{"plan": [...]}`，Worker 输出 `{"status", "result", "refs"}`，Aggregator 输出 `{"answer", "confidence"}`
- 适配器自动解析上游 JSON 提取摘要

### 4. 修复历史 Bug
- Token 双倍计算（v0.1-v0.4 一直存在）：`add` reducer + 节点累加 = 重复计数 → 改为节点只返回增量
- `allowed_tools: []` falsy 回退：`or` → `is not None`
- Redis Python 客户端版本冲突：从 8.x 降级到 4.x（兼容 Windows Redis 5.0）

## 六、已知限制

1. **实时流式降级**：SSE 事件仍在执行完成后批量 flush，未实现 `<500ms` 实时推送
2. **滑动窗口未集成**：模块已写好，但 OpenCode 单 session 单 prompt 的架构下暂无多轮对话可裁剪
3. **前端 Edit 模式不稳定**：部分配置项编辑后未正确保存到节点
4. **评测面板**：EvalPanel 有 ErrorBoundary 兜底但根因未修
